# Venue Strategy Policy

Use this reference for computer science venue targeting and positioning.

## Venue Record

Track in `docs/venue/venue-strategy.md`:

- venue
- track
- deadline
- page limit
- review model
- contribution types favored
- artifact expectations
- anonymity policy
- fit rationale
- risks
- required changes

## CS Venue Families

- ML/AI: NeurIPS, ICML, ICLR, AAAI, IJCAI
- NLP/IR/Web: ACL, EMNLP, NAACL, SIGIR, WWW
- HCI: CHI, CSCW, UIST
- Software engineering: ICSE, FSE, ASE
- Systems: SOSP, OSDI, NSDI, EuroSys
- Security/privacy: IEEE S&P, USENIX Security, CCS, NDSS
- Databases: SIGMOD, VLDB, ICDE
- Programming languages: PLDI, POPL, OOPSLA
- Theory: STOC, FOCS, SODA

## Fit Rule

Pick the venue whose reviewers are most likely to value the actual contribution,
not the venue with the highest prestige.
