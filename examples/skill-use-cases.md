# Skill Use Cases

These examples help reviewers and future maintainers understand when each skill
should trigger. They are compact boundary examples, not exhaustive prompts.

| Skill | Use Case | Near Miss |
|---|---|---|
| `academic-mcp-tooling` | Configure arXiv, DBLP, OpenAlex, Semantic Scholar, Zotero, or Overleaf MCP access for a research repository. | Summarize a paper that is already present in `sources/`. |
| `adversarial-peer-review` | Review a manuscript as a severe but fair conference reviewer. | Normalize BibTeX fields. |
| `artifact-open-science` | Prepare code, data, environment, and metadata for artifact evaluation or open-science release. | Brainstorm a new research question. |
| `citation-bibliography-tooling` | Clean BibTeX, citation keys, identifiers, duplicate references, and bibliography metadata. | Decide whether a cited claim is true. |
| `citation-claim-audit` | Verify claims, numbers, and citation statements against source evidence. | Convert a PDF to Markdown. |
| `cs-methodology-evaluation` | Design or audit CS baselines, metrics, datasets, ablations, and validity threats. | Pick a conference deadline. |
| `cs-venue-strategy` | Compare CS venues, tracks, fit, reviewer expectations, and artifact requirements. | Run or log an experiment. |
| `document-conversion` | Convert PDFs, DOCX files, scanned papers, tables, and figures into checked Markdown or assets. | Decide which papers belong in a SOTA review. |
| `ethics-data-governance` | Assess privacy, consent, license, scraping, retention, and sharing risks. | Polish manuscript prose. |
| `experiment-logbook` | Plan, run, compare, and record bounded experiments with command, config, seed, metric, and result. | Reorganize a messy source archive. |
| `paper-writing-review` | Draft or revise manuscript sections while preserving evidence, citations, and limitation honesty. | Install scholarly MCP servers. |
| `rebuttal-revision-strategy` | Map reviews into concede, defend, reframe, and revise actions for rebuttal or camera-ready work. | Build an initial literature matrix. |
| `repo-migration` | Move messy academic repositories, archives, proposals, notebooks, and outputs into the standard structure. | Choose statistical tests for cleaned results. |
| `research-data-analysis` | Clean data, run analysis, and produce publication-quality tables or figures. | Review a manuscript as a top-venue reviewer. |
| `research-design-positioning` | Turn a rough idea into research questions, hypotheses, contribution claims, scope, and evidence needs. | Normalize citation keys. |
| `research-project-maintenance` | Audit and repair repository structure, docs, wiki, logs, sources, outputs, and capability state. | Write a rebuttal response. |
| `research-project-router` | Select the right workflow when a task spans multiple research areas or the user is unsure which skill applies. | Perform a clearly scoped single-PDF source ingestion. |
| `research-repo-reproduction` | Reproduce or audit an external paper code repository, datasets, checkpoints, scripts, and paper claims. | Design a new research question for this project. |
| `research-ui-prototyping` | Build a research dashboard, annotation UI, metric browser, or evidence inspection interface. | Convert a scanned PDF to Markdown. |
| `skill-evaluation` | Pressure-test this skill package, trigger boundaries, references, install behavior, and validation. | Log a new experiment result. |
| `sota-literature-review` | Build a SOTA matrix, synthesize trends, compare methods, and identify gaps. | Prepare artifact evaluation packaging. |
| `source-ingestion` | Register PDFs, arXiv links, Zotero items, proposals, datasets, and metadata into the source ledger. | Choose a target venue. |
| `systematic-review-prisma` | Run a reproducible search, screening, inclusion/exclusion process, and PRISMA-style flow. | Write an informal related-work summary without screening. |
